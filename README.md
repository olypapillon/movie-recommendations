# Movie Recommendations

Semantic movie recommendations over the Netflix catalog. Each title is converted
into a compact textual representation, embedded into a vector with a local
embedding model, and indexed with [FAISS](https://github.com/facebookresearch/faiss)
for fast nearest-neighbor similarity search.

Describe a movie you like — its genre, cast, director, or just a plot summary —
and get back the most similar titles in the catalog.

![Streamlit app screenshot](docs/screenshot.png)

## How it works

1. **Represent** — each movie row is flattened into a short labeled text block
   (type, title, director, cast, year, genres, description). Blank fields are
   omitted. See `create_textual_representation` in `common.py`.
2. **Embed** — the text is sent to a local [Ollama](https://ollama.com) server
   running [`nomic-embed-text`](https://ollama.com/library/nomic-embed-text),
   which returns a **768-dimensional** float vector.
3. **Index** — all vectors are stored in a FAISS `IndexFlatL2` index (`train.py`)
   and persisted to a file named `index`.
4. **Query** — a search query is embedded with the *same* model and matched
   against the index by L2 distance, returning the closest titles.

## Embedding model

We use **`nomic-embed-text`** served via Ollama.

The original prototype used `llama2` for embeddings — a 7B-parameter *chat*
model that runs a full forward pass per title, taking ~1 minute per row on CPU.
A chat model is the wrong tool for embeddings. `nomic-embed-text` is a
purpose-built embedding model: much smaller, a fraction of a second per row on
CPU, and a drop-in swap via the same Ollama embeddings API. It outputs 768-dim
vectors (vs. llama2's 4096), which is why the FAISS index dimension is `768`.

> **Important:** any code that queries the index must embed the search text with
> the same `nomic-embed-text` model, or the query vector won't be comparable to
> the stored vectors.

## Requirements

- Python ≥ 3.12
- [uv](https://github.com/astral-sh/uv) for dependency management
- [Ollama](https://ollama.com) running locally
- `netflix_dataset.csv` in the project root

## Setup

```bash
# Install Ollama, then pull the embedding model
ollama pull nomic-embed-text

# Install Python dependencies
uv sync
```

## Usage

### 1. Build the index

Embeds every title in `netflix_dataset.csv` and writes the FAISS index to a file
named `index`:

```bash
uv run python train.py
```

### 2. Get recommendations

**Web app (Streamlit):**

```bash
uv run streamlit run app.py
```

Then open http://localhost:8501, fill in any fields describing a movie, and
click **Get recommendations**.

**Command line:**

```bash
uv run python reccomendation.py
```

You'll be prompted for each field; leave any blank to skip it.

## Project structure

| File                  | Purpose                                                        |
| --------------------- | -------------------------------------------------------------- |
| `common.py`           | Shared config, textual representation, and embedding helper.   |
| `train.py`            | Builds and persists the FAISS index from the dataset.          |
| `app.py`              | Streamlit web interface for recommendations.                   |
| `reccomendation.py`   | Command-line interface for recommendations.                    |
| `netflix_dataset.csv` | Source movie catalog.                                          |
| `index`               | Persisted FAISS index (generated by `train.py`).               |
